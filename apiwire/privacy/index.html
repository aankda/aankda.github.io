Privacy

Local by default. Clear about what moves.

API Wire handles request work on your device first. Optional sync only applies when you configure an endpoint and choose to use it.

Effective date: 2026-03-08
Local first

Data stays on your device

Collections, history, settings, and diagnostics stay local by default.

Optional sync

Cloud Sync only happens when you choose it

Sync is only active against an endpoint you configure yourself.

No default backend

No AANKDA sync service is published

This repository does not publish a default AANKDA-operated sync endpoint.

What stays on your device

By default, API Wire processes and stores request collections and history, local environment values and settings, and local diagnostics and log data on your device.

What is not sent by default

When you stay in local-only mode, API Wire does not send collections, workspace environment metadata, or local settings snapshots to a remote service by default.

Cloud Sync is optional

If you configure a sync endpoint and choose to use Cloud Sync, API Wire can transmit workspace snapshot collection data, workspace environment metadata, and selected local settings snapshots.

Secrets and secure values

Secrets are intended to be stored through OS keyring integrations. Secure values are not exported from workspace metadata as plaintext.

No default AANKDA backend

If you point Cloud Sync at an endpoint run by your organization or a third party, that operator's privacy terms apply separately.

Your control

You control the local data files stored by API Wire and can remove them from your device at any time.

Privacy questions can be sent to privacy@aankda.ai.

Privacy behavior changes?

This page changes when the behavior changes.
